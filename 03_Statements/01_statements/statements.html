Javaskript: Statements

Statement

Das Statement document.getElementById("demo").innerHTML = "Hallo Dolly!"; schreibt den String "Hello Dolly" in das Element mit der id="demo".

Ergebnis:

Reihenfolge der Statements

Die Anweisungen werden Zeile für Zeile nacheinander ausgeführt. Und zwar in der selben Reihenfolge, wie sie geschrieben wurden. 
						 				
<p>Ergebnis: <span id="ausgabe"><span></p>
<script>
	var x=42; var y=13; var z=x+y;
	document.getElementById('ausgabe').innerHTML = z;
</script>

Ergebnis:

In diesem Beispiel sind x, y und z Variablen. Mit der Variablen z wird das Ergebnis von x+y=42+13=55 ausgegeben.

Variationen des Quellcodes:

					
						 				
var x=42;  /* Das ist eine Variable*/
var y=13;  /* Blockkommentar*/
var z=x+y; // Zeilenkommentar
document.getElementById('ausgabe').innerHTML = z;

Code-Block

Beliebig viele Anweisungen lassen sich mit Schlüsselwort "function" innerhalb von geschweiften Klammern zu einer Anweisung zusammenfassen.

Beispiel: Wenn Sie den Button anklicken, dann ändert sich der Inhalt:

13 

					
						 				
<p><span id="first"></span><span id="second">13</span></p>
<p><button type="button" onclick="meineFunktion()">
   Versuch mal!
   </button>
</p>
function meineFunktion() {
	document.getElementById("first").innerHTML = "4";
	document.getElementById("second").innerHTML = "2";
}

Schlüsselwörter

Anweisungen in JavaScript starten oft mit Schlüsselwörtern. Hier eine kleine Liste von wichtigen Schlüsselwörtern:

break Beendet eine Schleife oder switch-Anweisung
continue Springt aus einer Schleife und startet ganz oben
debugger Beendet eine Ausführung in Javaskript und startet die Debugging-Funktion (wenn vorhanden!)
do ... while Schleife: Tue etwas, solange bis ...
for Schleife: Tue etwas, von ... bis
function Deklariert eine Funktion
if ... else Wenn-Dann-Bedingung für zwei Anweisungen
return Beendet eine Funktion mit einer Ausgabe
switch Case-Bedingung für beliebig viele Anweisungen
try ... catch Fehlerbehandlung
var Deklariert eine Variable
  • Am Ende einer Anweisung (=Statement) steht immer ein Semikolon ;
  • Anweisungen werden nacheinander ausgeführt.
  • Kommentare (/* ... */ oder // ...) sind notwendig!
  • Kommentare können auch zum Debuggen genutzt werden, indem Code-Zeilen oder Code-Blöcke auskommentiert werden.
  • Der Zuweisungsoperator = bewegt Werte (z.B. einer Variablen) von rechts nach links in einer Anweisung.
  • Linienlänge and Linienumbruch: Es ist üblich, nicht mehr als 80 Zeichen pro Programmierzeile zu verwenden. Wenn der Platz nicht reicht, darf umbrochen werden. TIPP: Umbrechen Sie nach dem Operator "="
  • Javaskript erlaubt Whitespaces (=Leerzeichen) im Quellcode, da diese von Javaskript ignoriert werden. Dadurch wird der Quellcode besser lesbar. Das ist aber Geschmacksache!
  • Beliebig viele Anweisungen lassen sich mit Schlüsselwort function innerhalb von geschweiften Klammern zu einer Anweisung zusammenfassen.
